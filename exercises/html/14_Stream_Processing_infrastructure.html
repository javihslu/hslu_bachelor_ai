Stream Processing Infrastructure

In this exercise we set up the event streaming service and get to know it. We use Redpanda as event streaming plattform. Redpanda describes their product as follows:
Redpanda is the most complete, Apache Kafka®-compatible streaming data platform, designed from the ground up to be lighter, faster, and simpler to operate. Free from ZooKeeper™ and JVMs, it prioritizes an end-to-end developer experience with a huge ecosystem of connectors, configurable tiered storage, and more.
For the exercise infrastructure, the simpler, lighter installation compared to the standard Apache Kafka from Confluent is especially important.

If you consume another message in the same way, you get the first message again. Kafka/Redpanda does not track who consumes which messages, and messages cannot be selectively deleted. This implementation as an append-only log is one of the reasons why the architecture scales so well horizontally.
Messages have a consumer offset, which is used to read messages from a specific point. It is the consumer's responsibility to track this offset or tell Kafka up to which offset was read.
High-level APIs do this automatically, when we use a low-level API like rpk, we must do this ourselves.
Look in the Redpanda Console for the offset of the second message and read exactly this one. The ways to specify offsets are described here.
Suggested solution
Du gibst den zu lesenden Offset mit --offset an: 
docker exec -it event-streaming rpk topic consume dev_sandbox --offset 1:2

Consumer groups

One concept we still need to know roughly is that of consumer groups. Through these, consumers can organize themselves into groups. Why consumer groups are necessary can be read here.
Important for us is that a message can be read by exactly one consumer per consumer group. If we read a message as part of a consumer group, we can (automatically) commit the offset of the message, so that this message is not read a second time afterwards.

Suggested solution
Here is a possible implementation using two functions. First, the training data is loaded from the object store. Then a kernel density estimator is fitted to the numerical columns to have a simple way to generate random but similar values. To generate the output, we sample the numerical columns. The categorical ones we draw randomly. 
from random import randrange
from sklearn.neighbors import KernelDensity


def setup_data():
    # fetch Training Dataset so we have a reference
    df = pd.read_parquet('s3://traindata/train_raw.parquet',
                            storage_options={"anon": False}).drop("class", axis="columns")

    # setup column names
    categoricals = ['cap-shape', 'gill-attachment', 'gill-color', 'stem-color']
    numericals = [c for c in df.columns if c not in categoricals]

    # fit an estimator to the numerical columns
    kde = KernelDensity()
    kde.fit(df[numericals])

    return kde, numericals, categoricals, df


def generate_event(kde, numericals, categoricals, df):
    # take one row so we have something to fill in our generated values
    new_row = pd.DataFrame(data=df.head(1))

    # generate one row
    new_row[numericals] = kde.sample(1)
    for col in numericals:
        # we are being lazy, when the kde yields a negative, just replace with mean
        if new_row[col][0] < 0:
            new_row[col] = df[col].mean()
    for col in categoricals:
        # for the categoricals, use a bounded random value
        new_row[col] = randrange(df[col].min(), df[col].max()+1)
    for col in new_row.columns:
        # make sure the datatypes match the reference
        new_row[col] = new_row[col].astype(df[col].dtype)

    return new_row.iloc[0].to_json()

import argparse
import sys
import time
from random import randrange


import pandas as pd
from kafka import KafkaProducer
from kafka.errors import KafkaError
from loguru import logger
from sklearn.neighbors import KernelDensity

producer = KafkaProducer(
    bootstrap_servers = "event-streaming:9092",
)

def on_success(metadata):
    logger.debug(f"Message produced to topic '{metadata.topic}' at offset {metadata.offset}")


def on_error(e):
    logger.error(f"Error sending message: {e}")


def setup_data():
    # fetch Training Dataset so we have a reference
    df = pd.read_parquet('s3://traindata/train_raw.parquet',
                            storage_options={"anon": False}).drop("class", axis="columns")

    # setup column names
    categoricals = ['cap-shape', 'gill-attachment', 'gill-color', 'stem-color']
    numericals = [c for c in df.columns if c not in categoricals]

    # fit an estimator the the numerical columns
    kde = KernelDensity()
    kde.fit(df[numericals])

    return kde, numericals, categoricals, df


def generate_event(kde, numericals, categoricals, df):
    # take one row so we have something to fill in our generated values
    new_row = pd.DataFrame(data=df.head(1))

    # generate one row
    new_row[numericals] = kde.sample(1)
    for col in numericals:
        # we are being lazy, when the kde yields a negative, just replace with mean
        if new_row[col][0] < 0:
            new_row[col] = df[col].mean()
    for col in categoricals:
        # for the categoricals, use a bounded random value
        new_row[col] = randrange(df[col].min(), df[col].max()+1)
    for col in new_row.columns:
        # make sure the datatypes match the reference
        new_row[col] = new_row[col].astype(df[col].dtype)

    return new_row.iloc[0].to_json()


def push_to_kafka(event, topic):
    # send to redpanda
    future = producer.send(topic=topic, key=b'key', value=event.encode('utf-8'))  # we skip message serializer and schema checks here
    future.add_callback(on_success)
    future.add_errback(on_error)
    producer.flush()


def run(topic, sleep_interval, burst_size, randomness):
    kde, numericals, categoricals, df = setup_data()

        for _ in range(burst_size):
            push_to_kafka(event=generate_event(kde, numericals, categoricals, df), topic=topic)

            sleep_milliseconds = sleep_interval + randrange(0, randomness*1000)
        except ValueError:
            # if randomness*1000 < 1
            sleep_milliseconds = sleep_interval
        time.sleep(sleep_milliseconds / 1000)
