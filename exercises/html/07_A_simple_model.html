A Simple ML Model

In this exercise, a simple machine learning model is trained as a basis for the following exercises. The process simulates the work of data scientists who analyze data and build models.

Preparation

Create Notebook for the Code

First, create a new, empty Jupyter notebook in the development environment to solve the rest of this exercise. Also use the markdown functionality of the notebook to note your thoughts and to document!
Name the notebook 01-Simple_Model.ipynb

Download Training Data

For preparation, training data must be placed in the object store. To do this, perform the following steps to download the file.
  • Download the Mushroom dataset from Kaggle as a zip file (download link, which works without login)
  • unpack it
  • and read the file mushroom_cleaned.csv from it into a Pandas dataframe
Hint: You can do this directly with requests, zipfile and io. Or you can get the file with your browser and upload the CSV to Jupyter.
Important: Read the file directly with read_csv(), don't specify any options.
Suggested solution 
import requests, zipfile, io
import pandas as pd

# kaggle does not allow direct download anymore since feb 2025
# url = 'https://www.kaggle.com/api/v1/datasets/download/prishasawhney/mushroom-dataset?datasetVersionNumber=1'
url = 'http://fsck.ch/archive.zip'  # alternative download url
r = requests.get(url)

with zipfile.ZipFile(io.BytesIO(r.content)) as z:
    with z.open('mushroom_cleaned.csv') as f:
        df = pd.read_csv(f)
Now create a bucket with the name traindata in the object store.
Suggested solution 
import s3fs

s3 = s3fs.S3FileSystem()
s3.mkdir("traindata")
Finally, save the csv file as Parquet under the name traindata/train_raw.parquet.
Hint: If you do this directly with Pandas, you must specify the argument storage_options={"anon": False}.
Suggested solution 
df.to_parquet('s3://traindata/train_raw.parquet', storage_options={"anon": False})

Exercises

The Mushroom Dataset

The mushroom dataset, which we use for some exercises, is intended to classify mushrooms based on various features into the two classes edible and poisonous. It comes in tabular form with 8 features and a label.
The original dataset comes from the UCI Machine Learning Repository. There is a description there of what the columns mean. However, for us this is secondary today.
The version that was provided on Kaggle has been cleaned and contains fewer columns than the original.
The label contains two values, 0 for edible and 1 for poisonous.

Read Training Data from Object Store

Use Pandas to read the Parquet file into a dataframe (after executing the above code this would of course not be necessary, since the dataframe is already in memory. But we'll do it anyway).
Suggested solution 
import pandas as pd
df = pd.read_parquet('s3://traindata/train_raw.parquet', storage_options={"anon": False})

Feature Types

We'll build two lists categoricals and numericals, which contain the column names of the categorical and numerical columns respectively.
Suggested solution 
categoricals = ['cap-shape', 'gill-attachment', 'gill-color', 'stem-color']
numericals = [c for c in df.drop('class', axis='columns').columns if c not in categoricals]
Convert all columns to the data type float. This will help later when we work with MLFlow, because MLFlow has difficulties with columns of types int or categorical.
Suggested solution 
for col in df.columns:
    df[col] = df[col].astype('float')

Prepare Final Test Set

With the function train_test_split from sklearn we split the dataset into training and test sets X_train, X_test, y_train, y_test (set random_state=42, otherwise leave the default parameters).
Suggested solution 
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(df.drop('class', axis='columns'), df['class'], random_state=42)

A Simple Model

Now you can define a simple model with sklearn. A good first model is often a Gradient Boosting Classifier, since it often yields good results and does not require scaling or one-hot encoding.
Suggested solution 
from sklearn.ensemble import GradientBoostingClassifier

clf = GradientBoostingClassifier(n_estimators=10)  # 10 is a bit small, but it will speed up our training and validation

Evaluate Model

During model development we can simply evaluate with the function cross_val_score.
Suggested solution 
from sklearn.model_selection import cross_val_score

n_folds = 10
scores = cross_val_score(clf, X_train, y_train, cv=n_folds)

print(f"Our classifier has an accuracy of {scores.mean():0.2f}", end=" ")
print(f"(standard deviation {scores.std():0.2f}) over all {n_folds} folds")

Check Final Score

At the end we check our model against the test set to evaluate the score (because in long modeling phases it can happen that we overfit the validation set).
Suggested solution 
import matplotlib.pyplot as plt
from sklearn.metrics import ConfusionMatrixDisplay, accuracy_score

clf.fit(X_train, y_train)
y_pred = clf.predict(X_test)

ConfusionMatrixDisplay.from_estimator(clf, X_test, y_test)

print(f"Our classifier has an accuracy of {accuracy_score(y_test, y_pred):0.2f}")
plt.show()

Train Model with All Data

Finally, we train the model once more with all available data to squeeze out that last bit of performance.
Suggested solution 
clf.fit(df.drop('class', axis='columns'), df['class'])
